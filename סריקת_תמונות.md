# סריקת תמונות ו-OCR

מערכת חילוץ נתונים מתמונות דוחות תאונה צבאיים (טופס 33.1106) בשילוב של OCR מקומי ו-Gemini Vision AI.

---

## סריקה מקומית — ocr-engine.js

### מבנה
- **~180 templates** של שדות טופס (שם, לוחית, תאריך, וכו')
- **Canvas-based matching**: השוואת patterns של תמונה אל template library
- **SRI hash** לאבטחה: `sha384-kEC+2KaGZ4b+M4g8HgCNH9N+2TfOMWcNR6Ttw3mclO4ppnH1tX4Xgl9jwfowxoxM`

### הפעולה
1. משתמש מצלם עמוד טופס
2. `ocr-engine.js` משווה לכל 180 templates
3. תוצאה: array of detected fields (undefined אם לא נמצא)
4. **תמיד מחזיר dict** — לעולם לא נכשל

## Gemini Vision AI

### מודלים
**Fallback chain:**
- `gemini-2.5-pro` (ראשי) — structured output, מדויק יותר
- `gemini-2.5-flash` (fallback בלבד) — מהיר, אם pro לא זמין/quota

**לא בשימוש:**
- `gemini-flash-latest` — מחזיר אנגלית במקום JSON (הוסר)

### Schema (_GEMINI_SCHEMA)
Structured output עם שדות נפרדים:
```json
{
  "date_day": "<number 1-31 or null>",
  "date_month": "<number 1-12 or null>",
  "date_year": "<number 2019-2100 or null>",
  "date_raw": "<raw date string as appears on form or null>",
  "license_plate": "<plate number or null>",
  "driver_name": "<driver full name or null>"
}
```

### System Instruction
טקסט הנחיה בעברית המבקש מ-Gemini:
- לחלץ תאריך מטופס 33.1106
- לוחית רישוי בפורמט XXX-XX-XXX,XXX-XXX  (חייב לוודא אם מדובר ברכב צבאי אז רק 6 ספרות אבל אם מדובר ברכב אזרחי אז זה 6 ספרות)
- שם נהג: שם פרטי + משפחה (2+ מילים עבריות)
- לדווח בפורמט JSON

### Retry Logic
- **429 quota exceeded**: 3 ניסיונות עם 10 שניות המתנה לפני fallback ל-flash
- **אחרות errors**: ישר ל-flash
- **Timeout**: fallback ל-flash

## שדות חילוץ ויולידציה

### תאריך
**חילוץ בעברית (_normalizeDate):**
- `date_day`, `date_month`, `date_year` → בנייה: `YYYY-MM-DD`
- `date_raw` → fallback אם parsing נכשל
- Cross-validation: אם `rawDate` סותר ל-day/month/year → `rawDate` גובר

**Regex (שורה 1806):**
⚠️ **בעיה ידועה**: `/(\\d{1,2})[\\s]*.../ ` בתוך regex literal = backslash+d, לא digit class.
- Last-resort ל-DD/MM ללא שנה תמיד מחזיר null
- **תיקון צריך:** `/(\d{1,2})[\s]*[\/.\-|\\lI][\s]*(\d{1,2})(?!\d)/`

### לוחית רישוי (_validatePlate)
**קטגוריות:**
- **צבאי**: 6 ספרות בדיוק (מינימום)
- **אזרחי**: `XXX-XX-XXX` (9 ספרות)
- **דחויים**: טלפונים (0505...), לא מספרים

**תוכנית**: "הפינה השמאלית עליונה" (בפרומפט)
- ⚠️ **בעיה ידועה**: בטופס RTL זה כנראה הפינה הלא נכונה
- **צריך לאמת** מול תמונה אמיתית של טופס 33.1106

### שם נהג (_validateName)
**כללים:**
- חובה: 2 מילים עבריות לפחות (שם פרטי + משפחה)
- מילה בודדת נדחית
- תווים מובנים: א-ת בלבד (לא ספרות, סימנים)

## ניקוי וצפייה בנתונים — data-processor.js

### BiDi Protection (שורה 43)
```javascript
// מוסיר U+202A–U+202E (RTL override characters):
let s = input.replace(/[\u202a-\u202e]/g, '');
```
מגן מפני Trojan Source attacks בטקסט עברי.

### cleanHebrew()
מטהר input עברי:
- מסיר בדים
- טיקון שגיאות עברית נפוצות
- Normalization (NFC)

### normalizePlate()
עיצוב לוחית: `XXX-XX-XXX` → לפרמט תקן

### normalizeDate()
פיתוח תאריך גולמי לפורמט קבוע (בדיקה קוודי validation)

## Debug Output

### Format
```
יום:X חודש:X שנה:X | גולמי:"..." | לוחית-גולמי:"..." לוחית-אחרי:"..." | נהג-גולמי:"..." נהג-אחרי:"..."
```

### דוגמה
```
יום:15 חודש:4 שנה:2026 | גולמי:"15.4.2026" | לוחית-גולמי:"123-45-678" לוחית-אחרי:"123-45-678" | נהג-גולמי:"David Cohen" נהג-אחרי:"David Cohen"
```

### ניתוח
- `לוחית-גולמי = —` → Gemini לא מצא כלום (בעיית פרומפט או תמונה)
- `לוחית-גולמי = מספר` אבל `לוחית-אחרי = —` → הוולידציה דוחה (format שגוי)
- **⚠️ בעיה ידועה:** debug לא מציג `parsedDate` לפני cross-validation
