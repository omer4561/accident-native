## 🔴 CRITICAL-1: סיסמת admin hardcoded

### הבעיה
```
index.html:2401 → const ADMIN_PASS = 'admin2025';
admin.html:117  → const ADMIN_PASS = 'admin2025';
```
הסיסמה גלויה בקוד כציבורי. כל מי שקורא את ה-source יכול להכנס.

### התיקון ✅
- **קבצים משונים:** `pages.yml`, `index.html`, `admin.html`
- **נוסחה build:** 
  ```bash
  ADMIN_HASH=$(echo -n "${ADMIN_PASS}acc2025" | openssl dgst -sha256 -hex | cut -d' ' -f2)
  sed -i "s|__ADMIN_PASS_HASH__|$ADMIN_HASH|g" www/index.html
  ```
- **בקוד:** סיסמה לא מופיעה. רק hash לפי `SHA-256(ADMIN_PASS + 'acc2025')`

### צעדים להפעלה
1. **הוסף GitHub Secret:**
   - Settings → Secrets and variables → Actions → New repository secret
   - Name: `ADMIN_PASS`
   - Value: `<הסיסמה החדשה שלך>`
2. **Push כלשהו ל-main** יריץ את `pages.yml` ויזריק את ה-hash

## 🔴 CRITICAL-2: OTP stored in localStorage with plaintext code

### הבעיה
```
index.html:660 → localStorage.setItem('pending_otp', { code: "123456", ... })
```
הקוד המלא של ה-OTP נשמר בצד הלקוח. כל extension/XSS קורא אותו ישירות.

### התיקון ✅
**בקוד:**
- הקוד חדש: **hash בלבד** → `SHA-256('otp:' + code)`
- `localStorage` → `{ codeHash: "a3f...", tries: 0, ... }`
- וידוא: הקוד הגולמי לא מאוחסן לעולם

**קבצים משונים:**
- `index.html` — שורות 642–707 (sendOTP, verifyOTP)

**עם rate limiting:**
- מקסימום 5 ניסיונות
- אחרי 5 ניסיונות → מחיקת pending_otp
- הודעה: "⛔ יותר מדי ניסיונות — שלח קוד מחדש"

### הבעיה
```

function _adminSessionValid() {
  const v = localStorage.getItem(ADMIN_AUTH_KEY) || '';
```

כל ערך ב-localStorage עובר. קל לעקוף בקונסול.

- `index.html:2418, 2465` 
- `admin.html:121, 149`

**פורמט token:** `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx:1234567890`

## 🟠 HIGH-1: XSS ב-admin panel

### הבעיה
```
index.html:2522 → ${meta.plate || 'לא זוהה'}  // No escaping!
                   ${meta.driver}, ${meta.userName} — אותו דבר
```
Meta fields מ-HuggingFace Dataset מוכנסים ישירות ל-innerHTML. תוקף שכותב דוח עם `<img src=x onerror=alert(1)>` יריץ XSS.

### התיקון ✅
**Escaping + URL whitelist:**
```js
// Helper function:
function _esc(s) {
  return String(s||'').replace(/&/g,'&amp;').replace(/</g,'&lt;').replace(/>/g,'&gt;').replace(/"/g,'&quot;').replace(/'/g,'&#39;');
}

// Usage:
${_esc(meta.plate) || 'לא זוהה'}
${_esc(meta.driver)}
${_esc(meta.userName)}

// Images — whitelist only HF URLs:
const safeUrls = (meta.urls || []).filter(u => 
  /^https:\/\/huggingface\.co\//.test(u)
);
```

**קבצים משונים:**
- `index.html:2420, 2515–2527`
- `admin.html:281–284`

## 🟠 HIGH-2: HF_TOKEN ב-URL כ-query parameter

### הבעיה
```
index.html:752 (old) → `${HF_FILES}/${path}?token=${HF_TOKEN}`
```
Token מופיע ב:
- Server logs
- Browser history
- HTTP Referrer headers
- Screenshots

### התיקון ✅
**Authorization header במקום:**
```js
// Old:
fetch(`${HF_FILES}/${p}?token=${HF_TOKEN}`)

// New:
fetch(`${HF_FILES}/${p}`, { 
  headers: { 'Authorization': `Bearer ${HF_TOKEN}` } 
})
```

**קבצים משונים:**
- `index.html:2463` (hfReadJson)
- `index.html:752` — הורדנו את `?token=` מה-URL הישיר

## 🟠 HIGH-3: סיסמת keystore hardcoded ב-build-apk.yml

### הבעיה
```bash
# Lines 74, 99:
KS_PASS: ${{ secrets.KEYSTORE_PASSWORD || 'AccidentApp2025!' }}

# Line 107:
ks_pass = os.environ.get('KS_PASS', 'AccidentApp2025!')
```
הסיסמה הקשוחה היא fallback אם ה-secret לא הוגדר. זה נחשף ב-CI logs ובקובץ build.gradle.

### התיקון ✅
הסרת ה-fallback hardcoded:
- שורה 74: `${{ secrets.KEYSTORE_PASSWORD }}` (ללא fallback)
- שורה 99: אותו תיקון
- שורה 107: `os.environ.get('KS_PASS', '')` (אין fallback)

**עכשיו הוא חובה להגדיר `KEYSTORE_PASSWORD` ב-GitHub Secrets.**
